Data

We will be getting the majority of our data from oddshark.com. This site will provide the odds data that we will use to make our application functional. Using the Puppeteer Package in JavaScript we have already been able to access relevant data for our project. Other data sources will be the users that sign up, game scores, and transaction history. We will be data scraping the website via javascript and python with an automated bot to scrape the data at different times of the day. This bot would then put the new data into our database.

They will then compare these scores with the contracts/bets to see which user won/lost. Christian will assess payout for the users based on results. Relations Scheme

  • Users(User_ID, name, email, balance, password) Su = {User_ID -> name, User_ID -> email, User_ID -> balance, User_ID ->password}
  • Bets(Bet_ID, Game_ID, sport, game_date, type) Sb = {Bet_ID -> Game_ID, Bet_ID -> sport, Bet_ID -> game_date, Bet_ID -> type}
  • Moneyline(Bet_ID, game_date, home_odds, away_odds) Sm = {Bet_ID -> game_date, Bet_ID -> home_odds, Bet_ID -> away_odds}
  • Spread(Bet_ID, game_date, home_odds, away_odds, line) Ss = {Bet_ID -> game_date, Bet_ID -> home_odds, Bet_ID -> away_odds, Bet_ID -> line}
  • OverUnder(Bet_ID, game_date, home_odds, away_odds, total) So = {Bet_ID -> game_date, Bet_ID -> home_odds, Bet_ID -> away_odds, Bet_ID -> total}
  • Contracts(Contract_ID, Bet_ID, User_ID, payout, bet_choice, listsed, listing_price, game_date, date_purcahsed, paidout) Sc = {Contract_ID -> Bet_ID, Contract_ID -> User_ID, Contract_ID -> payout, Contract_ID -> bet_choice, Contract_ID -> listsed, Contract_ID -> listing_price, Contract_ID -> game_date, Contract_ID -> date_purcahsed, Contract_ID -> paidout}
  • Games(Game_ID, home_team, away_team, completed, home_score, away_score, game_date) Sg = {Game_ID -> home_team, Game_ID -> away_team, Game_ID -> completed, Game_ID -> home_score, Game_ID -> away_score, Game_ID -> game_date}
